# 团队总览界面重构 - 变更说明

## 概述

按照 prompt.txt 的要求，对团队总览界面进行了彻底重构，采用更现代、更清爽的 SaaS 布局设计。

## 主要变更

### 1. 全局布局重构

**之前：**
- 左侧固定创建区（常驻）
- 右侧固定 Agent 面板（常驻）
- 中间团队卡片网格

**现在：**
- 顶部操作栏：左侧标题 + 右上角"+ 新建团队"按钮
- 主内容区：默认只显示团队卡片网格（1-3列响应式）
- 抽屉式 Agent 面板：仅在创建模式时从右侧滑出
- 轻量化拖拽区：仅在创建模式时显示在主内容区顶部

### 2. 创建模式交互流程

**触发方式：**
- 点击右上角"+ 新建团队"按钮进入创建模式

**创建模式界面：**
- 右侧滑出 Agent 列表面板（带遮罩层）
- 主内容区顶部显示轻量化拖拽接收区
- 拖拽区分为 Leader 区和成员区（左右并排）
- 提供"取消"按钮退出创建模式

**确认流程：**
- 拖拽完成后点击"下一步"
- 弹出居中 Modal 确认表单
- 填写团队名称（自动生成）和职责描述
- 点击"确认创建"完成

### 3. 团队卡片视觉重构

**设计规范：**
- 白底 (bg-white) + 柔和阴影 (shadow-sm hover:shadow-md)
- 微边框 (border border-slate-100)
- 大圆角 (rounded-2xl)
- 充足内边距 (p-6)

**卡片结构：**

**Header（头部）：**
- 左侧：团队名称（text-lg font-semibold text-slate-800）
- 右侧：三点菜单（编辑/删除操作）

**Body（核心成员区）：**
- Leader 展示：头像 + 皇冠图标 Badge
- 成员堆叠：头像重叠设计 (-space-x-2)，超出显示 +N

**Footer（底部状态栏）：**
- 极细分隔线 (border-t border-slate-100)
- 状态指示：彩色圆点 + 文字（绿点=活跃，灰点=空闲，黄点=阻塞）
- 次要信息：任务数、活跃时间（text-xs text-slate-400 + Lucide icons）

### 4. Agent 面板改造

**之前：**
- 固定在右侧，可折叠
- 宽度 360px（展开）/ 72px（折叠）
- 使用 localStorage 持久化折叠状态

**现在：**
- 抽屉式滑入（从右侧）
- 仅在创建模式时显示
- 带半透明遮罩层
- 点击遮罩或关闭按钮退出
- 固定宽度 380px

### 5. 创建区改造

**之前：**
- 固定在左侧，占据 420px 宽度
- 垂直布局（Leader 区在上，成员区在下）
- 较大的卡片容器

**现在：**
- 仅在创建模式时显示
- 居中显示，最大宽度 3xl
- 水平布局（Leader 区和成员区并排）
- 轻量化设计：浅色背景 + 虚线边框
- 确认表单改为 Modal 弹窗

## 设计亮点

1. **空间利用率提升**：默认状态下主内容区完全用于展示团队卡片，无常驻侧边栏占用空间

2. **交互流程优化**：创建团队从"常驻入口"改为"按需触发"，减少视觉干扰

3. **视觉质感提升**：
   - 卡片设计更精致（柔和阴影、微边框、大圆角）
   - 信息层次更清晰（Header/Body/Footer 分区明确）
   - 状态指示更直观（彩色圆点 + 图标）

4. **响应式设计**：团队卡片网格自适应 1/2/3 列布局

5. **动画效果**：
   - Agent 面板滑入/滑出动画
   - 创建区展开/收起动画
   - 确认表单 Modal 弹出动画
   - 卡片 hover 效果

## 兼容性说明

- 保持了原有的数据结构和 API 接口
- 保持了原有的拖拽功能（@dnd-kit/core）
- 保持了原有的团队创建流程逻辑
- 仅改变了 UI 布局和视觉样式
